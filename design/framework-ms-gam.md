# MS-GAM Framework: Current Main Blueprint

Updated: 2026-06-27

This file is the current control framework for the paper after the method shift to Markov-Switching GAM.

The old functional-GAM framework remains useful as background, but the main structure is now:

[
\text{latent regime}
\rightarrow
\text{regime-specific climate-response function}
\rightarrow
\text{functional geometry}
\rightarrow
\text{prediction under regime uncertainty}
\rightarrow
\text{structural robustness}
]

Important convention:

> All formulas are written for one destination. Different destinations are estimated separately, not as a pooled panel.

## 0. Core Position

The paper is no longer a simple GAM application.

It is a regime-dependent functional system paper:

> Tourism demand is generated by latent climate-response regimes. Each regime has its own smooth climate-response function, and the regime process evolves as a Markov chain.

The central empirical object is therefore:

[
\mathcal{F}^{(k)}(X_t)
]

not a single destination-level function.

## 1. Core Model

The latent regime is:

[
S_t \in \{1,\dots,K\}
]

with Markov transition:

[
P(S_t=j\mid S_{t-1}=i)=p_{ij}
]

For each latent regime \(k\), the climate-response function is:

[
\mathcal{F}^{(k)}(X_t)
=
s_k^{trend}(t)
+
s_k^{season}(m_t)
+
f_k^{temp}(T_t)
+
f_k^{rain}(R_t)
+
te_k(T_t,R_t)
+
f_k^{ext}(E_t)
]

The observation equation is:

[
y_t\mid S_t=k
\sim
\mathcal{N}
\left(
\mathcal{F}^{(k)}(X_t),
\sigma_k^2
\right)
]

where:

[
y_t=\log(Y_t+1)
]

See `design/ms-gam-core.md` for the compact formula note.

## 2. Role of Structural Breaks

Structural break analysis is not the main model.

Its role is diagnostic:

[
\text{breakpoints}
\Rightarrow
\text{visible evidence of regime instability}
]

The MS-GAM is the main structural model:

[
\text{MS-GAM}
\Rightarrow
\text{latent regime inference and regime-specific response functions}
]

So the narrative should not say that structural breaks and Markov switching are equivalent.

## 3. Chapter Logic

### Chapter 3: Data and System Design

Core job:

> Show that the data are asynchronous samples from destination-specific latent functional systems.

Needed adjustment:

[
X_t=(T_t,R_t,E_t)
]

should be described as the observed climate state entering a latent regime-dependent function.

Daily data still mainly support the construction of \(E_t\), frequency checks, and robustness tests.

### Chapter 4: MS-GAM Inference

Old role:

> reconstruct a smooth climate-response function via GAM.

New role:

> jointly infer latent regimes and regime-specific smooth response functions.

Chapter 4 should answer:

1. How is \(S_t\) modeled?
2. How is \(\mathcal{F}^{(k)}(X_t)\) estimated?
3. How are posterior regime probabilities obtained?

Key object:

[
P(S_t=k\mid y_{1:T})
]

This chapter should be renamed toward:

> Inference of Markov-Switching Climate-Response Functions

### Chapter 5: Regime-Specific Functional Geometry

Old role:

> compare functions across destinations.

New role:

> compare functions across regimes and destinations.

The functional object becomes:

[
\mathcal{F}^{(k)}
]

or, when comparing across destinations in text:

[
\mathcal{F}_i^{(k)}
]

with the reminder that these are separately estimated systems.

Distance and geometry should be written as:

[
d\left(\mathcal{F}_i^{(k)},\mathcal{F}_j^{(\ell)}\right)
]

rather than only:

[
d(\mathcal{F}_i,\mathcal{F}_j)
]

The chapter should answer:

1. Do climate-response functions differ across latent regimes?
2. Are high-risk regimes geometrically similar across destinations?
3. Does interaction geometry strengthen in some regimes?

### Chapter 6: Prediction under Regime and Extreme Climate Structure

Old role:

> test whether function features improve prediction.

New role:

> use MS-GAM as a structural representation system, then test whether regime-aware and extreme-aware representations improve downstream forecasting.

Core position:

> MS-GAM explains and extracts structure. The forecasting model \(g_\theta\) acts as an operational decoder of that structure.

#### 6.1 Structural representation layer

At forecast origin \(t\), MS-GAM produces filtered regime information:

[
\Pi_t
=
\left(
P(S_t=1\mid y_{1:t},X_{1:t}),\dots,P(S_t=K\mid y_{1:t},X_{1:t})
\right)
]

Define the forecast-side structural block as:

[
Z_t
=
\Psi(\Pi_t)
=
\left(
\Pi_t,
H_t^{\Pi},
\hat{S}_t
\right)
]

where

[
H_t^{\Pi}
=
-\sum_{k=1}^{K}\pi_{t,k}\log \pi_{t,k},
\qquad
\hat{S}_t=\arg\max_k \pi_{t,k}
]

Only filtered regime information may enter forecasting.
Smoothed posterior probabilities are excluded from the forecast layer because they use future data.

This layer is not the forecasting model itself. It is the state-representation layer.

#### 6.2 Forecasting model ladder

The core evaluation target is the one-step-ahead monthly forecast:

[
\hat{y}_{t+1\mid t}
]

Define the default forecast-side input blocks:

[
\mathcal{H}_t
=
\left(
y_t,\dots,y_{t-L+1}
\right)
]

[
\mathcal{C}_t
=
\text{forecast-available regular-climate block at origin }t
]

[
\mathcal{E}_t
=
\text{forecast-available extreme-weather block at origin }t
]

If ex ante climate forecasts are not used, \(\mathcal{C}_t\) and \(\mathcal{E}_t\) reduce to lagged climate blocks.
The same timing discipline must be imposed on the climate inputs of \(B_2\) and \(M_2\)-\(M_4\).

The ladder has two parts.

External anchors:

[
B_0:
\qquad
\hat{y}_{t+1\mid t}^{B_0}=y_{t+1-12}
]

[
B_1:
\qquad
\hat{y}_{t+1\mid t}^{B_1}
=
g^{SARIMA}(\mathcal{H}_t)
]

[
B_2:
\qquad
\hat{y}_{t+1\mid t}^{B_2}
=
g^{SARIMAX}(\mathcal{H}_t,\mathcal{C}_t)
]

Ablation sequence under a fixed learning engine:

[
g_{\theta^\star}
=
\arg\min_{g\in\mathcal{G}}CVErr_{M_4}(g)
]

where \(\mathcal{G}\) is a small candidate set of downstream forecasting models.
Once selected, the same \(g_{\theta^\star}\) is held fixed across all \(M\)-series rungs.

[
M_1:
\qquad
\hat{y}_{t+1\mid t}^{M_1}
=
g_{\theta^\star}(\mathcal{H}_t)
]

[
M_2:
\qquad
\hat{y}_{t+1\mid t}^{M_2}
=
g_{\theta^\star}(\mathcal{H}_t,\mathcal{C}_t)
]

[
M_3:
\qquad
\hat{y}_{t+1\mid t}^{M_3}
=
g_{\theta^\star}(\mathcal{H}_t,\mathcal{C}_t,\mathcal{E}_t)
]

[
M_4:
\qquad
\hat{y}_{t+1\mid t}^{M_4}
=
g_{\theta^\star}(\mathcal{H}_t,\mathcal{C}_t,\mathcal{E}_t,Z_t)
]

Each \(M\)-series increment corresponds to an added information layer:

[
\text{demand history}
\rightarrow
\text{regular climate}
\rightarrow
\text{extreme climate}
\rightarrow
\text{filtered regime structure}
]

#### 6.3 Extreme climate as structural perturbation

Extreme climate is not just another covariate. It should be framed as a perturbation mechanism:

[
E_t
\Rightarrow
\begin{cases}
\text{shift regime transition tendency}\\
\text{amplify regime-specific response functions}\\
\text{distort tail behavior of }y_t
\end{cases}
]

The \(B\)-series are fixed external reference points, not lower rungs in the same ablation sequence.
The chapter should therefore read the ladder through explicit contrasts, not only through average forecast error.

Key diagnostic targets:

[
\text{Gain}_{climate}
:
M_2
\text{ vs. }
M_1
]

[
\text{Gain}_{extreme}
:
M_3
\text{ vs. }
M_2
]

[
\text{Gain}_{structure}
:
M_4
\text{ vs. }
M_3
]

The \(M_4\) versus \(M_3\) contrast is the sharpest structural-validation test.
It holds the downstream learning engine and conventional inputs fixed and adds only filtered regime structure.

#### 6.4 Forecast horizon discipline and role of \(g_\theta\)

One-step-ahead forecasting remains the core design.
Recursive multi-step forecasting is a secondary extension.

For \(h\geq 2\),

[
\hat{y}_{t+h\mid t}^{M_j}
=
g_{\theta^\star}
\left(
\widehat{\mathcal{H}}_{t+h-1\mid t},
\mathcal{I}_{t}^{(j,h)}
\right)
]

where \(\widehat{\mathcal{H}}_{t+h-1\mid t}\) updates the demand-history block recursively with previously predicted values, and \(\mathcal{I}_{t}^{(j,h)}\) denotes the forecast-available information set associated with rung \(M_j\).

The forecasting model is not a replacement for MS-GAM.
It acts as the operational decoder of the structural information extracted by MS-GAM.

### Chapter 7: Structural Robustness

Old role:

> test whether the estimated function space is stable.

New role:

> test whether regime inference, regime-specific functions, and regime-aware prediction are stable.

Three robustness targets:

[
P(S_t=k\mid y_{1:T})
]

[
\mathcal{F}^{(k)}(X_t)
]

[
\hat{y}_{t+1}
]

Main checks:

1. regime stability across specifications
2. function stability within regimes
3. daily-monthly aggregation invariance
4. prediction-gain stability

## 4. Extreme Weather Term

The interaction surface and extreme-weather exposure should stay conceptually separate.

[
te_k(T_t,R_t)
]

captures the regime-specific continuous interaction surface between temperature and rainfall.

[
f_k^{ext}(E_t)
]

captures explicit extreme-weather exposure.

Candidate \(E_t\) definitions:

[
E_t^{bin}
]

binary event exposure.

[
E_t^{int}
]

continuous tail intensity.

[
E_t^{comp}
]

compound extreme exposure.

The final \(E_t\) construction should be fixed before empirical notebooks begin.

[
S_t
\rightarrow

\rightarrow
\Phi(\mathcal{F}^{(k)})
