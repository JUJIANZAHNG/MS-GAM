# MS-GAM Framework: Main Blueprint

Status: control file
Updated: 2026-06-30

This file is the current control blueprint for the paper after the method shift to Markov-Switching GAM. It is the chapter-level control document. The canonical estimator is specified in `design/ms-gam-core.md`. Other chapter files are referenced from this file.

Important convention:

> All formulas are written for one destination. Different destinations are estimated separately, not as a pooled panel.

## 0. Core position

The paper is a regime-dependent functional system paper. Tourism demand is generated by latent climate-response regimes. Each regime has its own smooth climate-response function, and the regime process evolves as a Markov chain. The central empirical object is the regime-specific function F^{(k)}(X_t), not a single destination-level function.

## 1. Top-level chain

The top-level chain is:

MS-GAM inference (Chapter 4)
  → Chapter 5: regime-specific functional geometry branch
  → Chapter 6: forecasting and structural validation branch

Chapters 5 and 6 are parallel downstream consumers of the MS-GAM output. They are not required to be run in a strict serial order. Chapter 5 consumes the regime-specific functions F̂^{(k)}. Chapter 6 consumes the filtered structural representation Z_t. Each branch has its own core object and its own downstream analysis.

## 2. Core model

The canonical baseline model is specified in `design/ms-gam-core.md`. Summary:

State process — first-order homogeneous Markov chain:

S_t ∈ {1, ..., K},   P(S_t = j | S_{t-1} = i) = p_{ij}

Observation equation — current-period monthly block only:

F^{(k)}(X_t) = s_k^{trend}(t) + s_k^{season}(m_t) + f_k^{temp}(T_t) + f_k^{rain}(R_t) + te_k(T_t, R_t) + f_k^{ext}(E_t)

y_t | S_t = k ~ N(F^{(k)}(X_t), σ_k^2)

with y_t = log(Y_t + 1).

Lagged climate memory does not enter the baseline. The extreme term E_t does not enter the transition equation under the baseline. See `design/ms-gam-core.md` for the canonical specification.

## 3. Chapter logic

### Chapter 1: Introduction

Motivate the nonlinear, regime-dependent climate-tourism system.

### Chapter 2: Conceptual Framework

The conceptual layer. Defines the regime-dependent functional system, the climate-memory operator M_L(X_t), and the three conceptual roles of extremes. Adds a claim boundary that the current baseline uses a homogeneous transition and the current-period monthly block. See `design/method.md`.

### Chapter 3: Data and System Design

The observational foundation. Defines separate destination-level estimation, monthly core frequency, daily support role, asynchronous destination-level sampling, and the role of external destinations as external validation. See `design/data.md`.

### Chapter 4: MS-GAM Inference

The inference bridge. Delivers regime-specific functions F̂^{(k)}, filtered probabilities Π_t, smoothed probabilities, and the transition matrix P̂. See `design/function-generate.md` and `design/ms-gam-core.md`.

### Chapter 5: Regime-Specific Functional Geometry

Core object: regime-specific climate-response geometry F_r^{(k)}.

This chapter is a comparison branch. It consumes F̂^{(k)} from Chapter 4 and compares regime-specific response objects across regimes and destinations:

- distance and geometry between regime-specific functions d(F_i^{(k)}, F_j^{(ℓ)})
- identification of high-risk regimes across destinations
- characterization of interaction geometry across regimes

Regime labels are local to each destination: "regime 1" in destination i is not, a priori, the same structural state as "regime 1" in destination j. Cross-destination comparison therefore requires a regime alignment rule. The primary (baseline) rule is within-destination severity ordering; the stronger rule is feature-based matching on Φ_r^{(k)}. See `design/functional-comparision.md` §2 and `design/outline.md` §5.4.1 for the full rule and its reporting discipline.

Φ_r^{(k)} denotes the destination-specific geometric or functional summary of a regime-specific function used for comparison. It belongs to the Chapter 5 comparison branch. It is not a Chapter 6 baseline input.

### Chapter 6: Prediction under Regime and Extreme Climate Structure

Core object: filtered structural representation Z_t.

This chapter is the forecasting and structural validation branch. It consumes Π_t from Chapter 4 and builds the forecast-side structural block:

Z_t = Ψ(Π_t) = (Π_t, H_t^Π, Ŝ_t)

where H_t^Π is the entropy of Π_t and Ŝ_t is the MAP regime. The forecasting model g_{θ*} is the operational decoder of Z_t.

The forecast ladder is M_1 through M_4, with B_0, B_1, B_2 as fixed external anchors. The M_4 vs. M_3 contrast is the primary structural-validation contrast. The chapter is focused on information gain, not on a model horse race. See `design/prediction.md`.

Chapter 6 is not a second independent forecasting paper. It uses MS-GAM as a structural representation system and tests whether the structure-aware representation improves downstream forecasting.

### Chapter 7: Structural Robustness

Tests stability of regime inference, regime-specific functions, and prediction gains. Consumes P̂, F̂^{(k)}, and the prediction-gain table from earlier chapters.

### Chapter 8: Discussion

Interpret tourism systems as regime-conditioned climate-response geometries.

### Chapter 9: Conclusion

Structural and predictive findings.

## 4. Extreme weather term

The interaction surface te_k(T_t, R_t) and the extreme term f_k^{ext}(E_t) carry distinct conceptual loads:

- te_k(T_t, R_t) captures the regime-specific continuous interaction between temperature and rainfall.
- f_k^{ext}(E_t) captures explicit extreme-weather exposure.

These are not substitutes for each other. The final E_t construction is fixed in the empirical notebooks, not in this blueprint.
