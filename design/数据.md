# 📐 Chapter 3 — Data and System Design as Function Sampling

------

# 3. Data as samples from a latent functional system

We assume tourism demand is generated by a latent function system:

# [ Y_{r,t}

f_{r,\mathcal{R}_r(t)}(\mathcal{M}_L(\mathcal{X}*t))
+
\varepsilon*{r,t}
]

However, this function is not directly observable.

Instead, we observe:

> discrete samples of a continuous functional process.

------

## 🔑 key idea：

Chapter 3 = **sampling theory of function space**

------

# 3.1 Multi-system structure

Define set of destinations:

[
\mathcal{R} =
\mathcal{R}^{domestic}_1
\cup
\mathcal{R}^{domestic}_2
\cup
\mathcal{R}^{external}
]

where:

- Domestic system I: Hong Kong / Macau / Taiwan
- Domestic system II: West Lake / Jiuzhaigou / Gulangyu
- External system: Singapore / Hawaii

------

## interpretation:

Each ( r \in \mathcal{R} ) is:

> one realization of a function generator ( f_r )

not a “panel unit”

------

# 3.2 Temporal sampling structure

Define observation grid:

[
t \in \mathcal{T}_r
]

where:

- monthly observations (main structure)
- daily observations (subset systems)

------

## key issue: irregular sampling

We explicitly allow:

[
|\mathcal{T}_r| \neq |\mathcal{T}_s|
]

and:

[
\mathcal{T}_r \not= \mathcal{T}_s
]

------

👉 meaning:

> this is not panel data, but **asynchronous functional sampling**

------

# 3.3 Climate observation space

Define climate state:

[
\mathcal{X}*{r,t} = (T*{r,t}, P_{r,t}, E_{r,t})
]

where:

- temperature
- precipitation
- extremes

------

## aggregation rule (important for your design)

Monthly aggregation is defined as operator:

[
\mathcal{A}:
{x_{r,d}}*{d \in month}
\rightarrow x*{r,t}
]

but we preserve:

- mean
- variance
- extremes
- frequency

So:

# [ x_{r,t}

(
\mu_T, \mu_P, \sigma_T, \text{extreme count}, \dots
)
]

------

👉 this is crucial:

> aggregation is not lossless compression, but structured projection

------

# 3.4 Dual-frequency design (key methodological pillar)

We define two observation modes:

------

## (1) high-frequency system

[
\mathcal{T}_r^{daily}
]

used for:

- Macau
- Jiuzhaigou

------

## (2) low-frequency system

[
\mathcal{T}_r^{monthly}
]

used for:

- Hong Kong
- Taiwan
- etc.

------

## key implication:

We explicitly test:

[
f_r^{daily} \overset{?}{\approx} f_r^{monthly}
]

------

👉 this becomes your:

> aggregation invariance test (Chapter 7 link)

------

# 3.5 Cross-system heterogeneity assumption

We do NOT assume:

[
f_r = f_s
]

or shared parameters.

Instead:

[
f_r \sim \mathcal{F}
\quad \text{(function distribution)}
]

Each destination is a:

> sample from a function-valued distribution

------

# 3.6 External systems as domain shift test

Define:

[
\mathcal{R}^{external} = {Singapore, Hawaii}
]

We assume:

[
f_{external} \not\sim f_{domestic}
]

but we test:

> whether structural representation generalizes

------

👉 this sets up Chapter 6 (transferability)

------

# 3.7 Data as noisy function observations

Final observation equation:

# [ Y_{r,t}

f_r(\mathcal{X}*{r,t})
+
\varepsilon*{r,t}
]

where:

- (f_r): latent function
- (\varepsilon): noise + unobserved shocks

------

## key reinterpretation:

| classical view  | your view             |
| --------------- | --------------------- |
| regression data | function samples      |
| panel data      | function realizations |
| time series     | trajectory sampling   |

------

# 3.8 Data generating hierarchy (bridge to Chapter 4)

We define full hierarchy:

[
\mathcal{X}*{r,t}
\rightarrow
f_r(\cdot)
\rightarrow
Y*{r,t}
]

but we only observe:

[
{(\mathcal{X}*{r,t}, Y*{r,t})}
]

and must reconstruct:

[
f_r
]

------

## closure of Chapter 3:

[
\boxed{
\text{Observed data}
\Rightarrow
\text{noisy samples of latent function system}
}
]
