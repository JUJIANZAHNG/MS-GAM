# Unified Ontology Diagram after the MS-GAM Shift

Updated: 2026-06-28

## 1. Figure purpose

This note controls the logic of Figure 1 after the move to a regime-dependent functional system.

The diagram should no longer center on one static function \(f_r(x)\).
It should center on a destination-specific latent regime system that yields both regime-specific functions and forecast-side structural representations.

Suggested title:

> Unified Ontology of Regime-Dependent Climate-Tourism Functional Systems

## 2. Four theory inputs

The conceptual figure still draws from four theory blocks:

| Theory block | What it contributes |
|---|---|
| Functional Data Analysis | function-valued object view |
| GAM / semiparametric estimation | smooth response reconstruction |
| state-space / Markov switching | latent regime evolution |
| predictive learning | downstream decoding of structural information |

These are not parallel add-ons. They describe different layers of the same destination-level system.

## 3. Central object

The central object should be written as:

[
\boxed{
\left(
S_{r,t},
f_{r,S_{r,t}}(\mathcal{M}_L(X_t))
\right)
}
]

This says the system has two inseparable pieces:

- a latent regime process \(S_{r,t}\)
- a regime-dependent climate-response function \(f_{r,S_{r,t}}(\cdot)\)

## 4. Layered diagram logic

### Level 1: Functional object space

[
\mathbb{F}_r
=
\{f_{r,1},\dots,f_{r,K_r}\}
]

Meaning:

> each destination is represented by a family of regime-specific response functions rather than one fixed curve.

### Level 2: MS-GAM estimation layer

[
(y_{1:T},X_{1:T})
\rightarrow
\left(
\hat{\mathcal{F}}_r^{(1:K)},
\Pi_{1:T},
\hat{\mathbf{P}}_r
\right)
]

Meaning:

> the estimation layer jointly recovers smooth response functions, regime probabilities, and transition dynamics.

### Level 3: Functional geometry layer

[
\hat{\mathcal{F}}_r^{(k)}
\rightarrow
\Phi_r^{(k)}
]

where:

[
\Phi_r^{(k)}
=
\left(
C_{r,k}^{temp},
C_{r,k}^{rain},
G_{r,k}^{int},
A_{r,k}^{ext}
\right)
]

Meaning:

> regime-specific functions are projected into measurable geometric summaries.

### Level 4: Structural representation layer

[
\Pi_t
\rightarrow
Z_t
=
\left(
\Pi_t,
H_t^{\Pi},
\hat S_t
\right)
]

Meaning:

> only filtered regime information is allowed to enter forecasting.

### Level 5: Downstream forecasting layer

[
\left(
\mathcal{H}_t,
\mathcal{C}_t,
\mathcal{E}_t,
Z_t
\right)
\rightarrow
g_{\theta^\star}
\rightarrow
\hat y_{t+1\mid t}
]

Meaning:

> forecasting is the downstream decoder of structural information, not the place where MS-GAM is replaced.

## 5. Closed-loop system view

The full ontology can be drawn as:

```text
Observed climate and demand
          ↓
       MS-GAM
          ↓
 latent regimes + regime-specific functions
          ↓
 functional geometry Φ and structural block Z_t
          ↓
 downstream forecast g_θ
          ↓
 out-of-sample validation and robustness
```

This is the paper's real closed loop.

## 6. Compact mathematical form

The figure-level summary can be written as:

[
\boxed{
(y_{1:T},X_{1:T})
\Rightarrow
\left(
\hat{\mathcal{F}}_r^{(1:K)},
Z_t
\right)
\Rightarrow
\hat y_{t+1\mid t}
}
]

This is cleaner than centering the figure on a regime-free single function.
